隐写基础认知

图像隐写核心概念

将信息（flag、密钥等）隐藏在图像文件中，肉眼不可见，但可通过特定方法提取，关键在于3点

1不影响原始图像的视觉质量

2抵抗格式转换、压缩等操作

3能隐藏足够多的数据

根据不同的图片载体一般有不同的隐写方式、特点和提取工具

BMP 无压缩，直接修改像素 Stegsolve、Python PIL

PNG 支持透明度，多通道存储 zsteg、pngcheck

JPEG 有损压缩，频域隐藏 Steghide、jsteg

GIF 多帧动画，利用颜色表 GifFrame、ffmpeg

基础隐写技术

LSB隐写

原理：修改像素RGB值的最后1-2个bit（人类视觉不敏感）

from PIL import Image

def lsb_encode(img_path, message):

img = Image.open(img_path)

binary_msg = ''.join(format(ord(c), '08b') for c in message)

pixels = img.load()

idx = 0

for x in range(img.width):

    for y in range(img.height):

        r, g, b = pixels[x, y]

        # 修改R通道的LSB

        if idx < len(binary_msg):

            r = (r & 0xFE) | int(binary_msg[idx])

            idx +=1

        pixels[x, y] = (r, g, b)

img.save('encoded.png')

通过如上代码可以实现隐写

检测方法方法有：

Stegsolve：使用Analyse > Data Extract查看位平面

统计学检测：观察像素值分布是否符合自然图像规律

文件附加

原理：在图像文件末尾追加数据（如ZIP、文本等）

实战中可以用dd命令分离附加数据

dd if=image.jpg bs=1 skip=<原文件大小> of=hidden.zip

或者用binwalk直接进行扫描

binwalk -e image.jpg

元数据隐写（EXIF/IPTC）

写入数据

exiftool -Comment=”flag{secret}” image.jpg

提取注释

exiftool image.jpg | grep Comment

文件格式分析

PNG深度解剖

文件结构：

文件头(89 50 4E 47 0D 0A 1A 0A)

IHDR（宽高/位深）

PLTE（调色板）

IDAT（压缩的像素数据）

IEND（结束块）

IDAT块压缩：使用zlib解压后的数据可能隐藏额外信息

import zlib

with open(‘image.png’, ‘rb’) as f:

data = f.read()

idat_start = data.find(b'IDAT') + 4

idat_data = zlib.decompress(data[idat_start:])

print(idat_data.hex())  # 查看原始像素数据

CRC校验绕过：修改数据后重新计算CRC32值

JPEG的DCT隐写

流程分解：

色彩空间转换：RGB → YCbCr

分块离散余弦变换（DCT）：8x8块转换为频域系数

量化：除以量化表并四舍五入

熵编码：Huffman编码存储

Jsteg算法：

在量化后的非零AC系数的最低有效位嵌入数据

检测工具：jsteg reveal image.jpg

高级隐写

频域隐写

DWT小波变换步骤：

将图像分解为LL（低频）、LH、HL、HH（高频）子带

在HH子带嵌入数据（高频噪声不易察觉）

实现方法：

使用OpenStego进行DWT隐写

openstego embed -mf secret.txt -cf cover.jpg -sf stego.jpg

JPEG双压缩隐写

JPEG压缩分为多个步骤：

色彩空间转换：RGB → YCbCr（分离亮度和色度）

分块DCT变换：将8x8像素块转换为频域系数（DC低频 + AC高频）

量化：用 量化表（Q-table） 对DCT系数取整（主要信息损失步骤）

熵编码：使用Huffman编码压缩数据

双压缩隐写原理

对已经压缩过的JPEG图片再次进行压缩（可能修改量化表或隐写数据）

隐写者先解码原始JPEG然后嵌入数据最后用新量化表重新压缩

直接对JPEG文件进行二次保存

2双压缩痕迹的形成

量化步长差异：两次压缩使用的量化表不同

DCT系数取整残留：

第一次量化：DCT系数变为 C1 = round(DCT / Q1)

第二次量化：C2 = round(C1 * Q1 / Q2)

当Q2 > Q1时，导致系数分布异常

攻击者可在二次压缩时修改量化表或DCT系数来携带信息

双压缩本身是常见操作（如编辑后保存），不易引起怀疑

可以用工具对双压缩进行检测

jpegdump -q stego.jpg # 显示当前量化表

jpeginfo -cqt stego.jpg # 检查量化表是否标准
